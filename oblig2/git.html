Git & GitHub

Git er et verktøy for fildeling og versjonskontroll. Git er supert for gruppeprosjekter hvor alle kan laste ned siste versjon av prosjektet, og lett jobbe sammen fra forskjellige steder i verden. Med Git kan man gå tilbake i historikken til prosjektet, og se hva som har blitt endret, og deretter laste ned tidligere versjoner. Dette er praktisk hvis noe har gått galt og du vil gå tilbake og endre. Man kan også lage flere versjoner av et program for publisering.

GitHub

GitHub er en tjeneste som tilbyr Git som versjonskontroll. Med andre ord, Git og GitHub er to forskjellige ting. Git er som er nevnt et verktøy for fildeling og versjonskontroll. GitHub tilbyr brukere denne tjenesten. På GitHub kan du laste opp prosjekt mappen din og starte et prosjekt. Her kan alle klone mappen din og se koden din. Du kan velge å gjøre prosjektet "public" eller "private". Dette gjør det enklere å jobbe sammen med andre på samme prosjekt. Du får alltid den siste versjonen av prosjektet og du kan enkelt laste opp dine endringer til GitHub, så andre har tilgang til dem.

Eksterne Verktøy

Det finnes flere eksterne programmer som gjør det enklere og mer effektiv bruk av Git. Jeg Bruker GitKraken, som gir deg muligheten til å "push'e" nye filer direkte til GitHub lokalt fra din maskin. Du kan også lett klone en GitHub mappe ved hjelp av GitKraken.
GitKraken lager også det vil kaller "branshes" eller grener på godt norsk. Dette visualiserer endringene som blir gjort. Dette vises best når flere personer jobber på samme prosjekt. Man kan da lage nye grener for kun endringer som blir gjort i CSS, HTML, Script etc...
Med andre ord GitHub er fantastisk!
